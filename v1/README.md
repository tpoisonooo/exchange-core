  为什么有效
  1. 消除 CPU 超订阅：基线配置下 4 ME + 2 RE + GroupingProcessor + ResultsHandler 等线程总数超过 8 核，导致 AffinityThreadFactory 频繁出现 No reservable CPU，大量线程
     pin 到 cpu -1。
  2. BUSY_SPIN 策略不再空转：线程能稳定绑定到独立核心后，Disruptor 的 BUSY_SPIN 等待策略避免了上下文切换开销，CPU 周期真正用于处理订单。
  3. 无核心代码改动：仅调整测试配置，零副作用，不会影响其他功能测试或引擎逻辑
