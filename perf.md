## 2. 实测性能基线 (exchange-core)

测试时间：2026-04-14  
测试项：吞吐量 (Throughput) + 延迟 (Latency)

### 2.1 吞吐量

- **平均吞吐：4.87 MT/s**
- 峰值约 5.42 MT/s，稳定区间 4.8–5.4 MT/s
- `MT/s` = Million Transactions / second（百万笔/秒）

### 2.2 延迟

| 负载 | P50 (中位数) | P99 | P99.9 | 状态说明 |
|------|-------------|-----|-------|----------|
| 0.2 MT/s | **0.85 µs** | 13.2 ms | 24.0 ms | 极低中位数 |
| 1.0 MT/s | **0.84 µs** | 17.3 ms | 30 ms | 中位数仍 < 1 µs |
| 2.0 MT/s | **1.09 µs** | 65 ms | 71 ms | 尾部延迟开始放大 |
| 3.0 MT/s | **1.14 µs** | 1.52 ms | 1.64 ms | 不同测试轮次数据 |
| 3.5 MT/s | **7.6–11.0 µs** | 20–41 ms | 22–43 ms | 接近吞吐上限，抖动明显 |
| 3.6 MT/s | **22.0 ms** | 154 ms | 164 ms | **过载，延迟急剧恶化** |

> **规律**：在吞吐未达到饱和（约 3.5 MT/s）前，**P50 中位数延迟始终保持在 ~1 µs**；超过饱和点后，延迟急剧上升。

## 3. CPU 理论峰值参数

### 3.1 物理 CPU 单颗完整规格 (Intel Xeon Platinum 8369B)

| 指标 | 理论峰值 |
|------|----------|
| 物理核心 | 64 Cores / 128 Threads |
| FP64 峰值 | 64 × 3.5 GHz × 32 = **~7.17 TFLOPS** |
| FP32 峰值 | 64 × 3.5 GHz × 64 = **~14.34 TFLOPS** |
| 内存带宽 | 8 通道 DDR4-3200 = **204.8 GB/s** |
| UPI 互联 | 3× 11.2 GT/s |
| L3 Cache | ~1.5 MB/core × 64 = **~96 MB** |

### 3.2 当前实例 (8 vCPU / 4 物理核) 按比例折算

| 指标 | 当前实例估算 |
|------|-------------|
| FP64 峰值 | 4c × 3.5 GHz × 32 ≈ **448 GFLOPS** |
| FP32 峰值 | 4c × 3.5 GHz × 64 ≈ **896 GFLOPS** |
| AVX-512 整数寄存器吞吐 | 4c × 3.5 GHz × 64 bytes ≈ **896 GB/s** |
| 实际可用内存带宽 | 云实例共享通道，估算约 **20–60 GB/s** |

## 4. 分析结论

- `exchange-core` 属于 **内存延迟敏感型** workload（撮合引擎），其瓶颈通常不是浮点算力，而是：
  - **L1/L2/L3 缓存命中率**
  - **内存访问延迟**（Ice Lake 本地内存延迟约 **85 ns**）
  - **单核主频** 和 **CPU 亲和性调度**

- 在当前 **8 vCPU** 实例上，exchange-core 达到 **~4.87 MT/s** 的平均吞吐，低负载下 **P50 延迟 ~0.8–1.3 µs**，这是一个非常优秀的表现，符合 HFT（高频交易）级别低延迟撮合引擎的设计目标。
